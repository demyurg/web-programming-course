LR12: PWA & Offline-first

Progressive Web Apps для реальной сети

Цель: сделать приложение устанавливаемым (installable), быстрым и устойчивым к offline-сценариям.

План занятия

  1. PWA концепция и offline-first
  2. Service Worker и жизненный цикл (lifecycle)
  3. Кэш-стратегии и fallback
  4. Offline data + sync
  5. Manifest и установка
  6. Проверка в DevTools/Lighthouse

Что такое PWA

  • Веб-приложение с нативными привычками
  • Устанавливаемое (installable) через браузер
  • Работает лучше при нестабильной сети

PWA = не только иконка, но и надежный offline-сценарий.

Почему offline-first важен

Интернет нестабилен. Пользователь оценивает приложение, а не качество сети.
  • Снижаем риск потери данных
  • Держим предсказуемый UX
  • Уменьшаем фрустрацию при обрывах

Критерии PWA

  • Installable - устанавливаемое
  • Fast - быстрое
  • Reliable - надежное
В LR12 мы целимся именно в эти три критерия.

Service Worker: роль

  • Перехватывает запросы
  • Управляет кэшем
  • Отдает fallback при offline

Работает только в HTTPS/localhost и не управляет DOM напрямую.

Жизненный цикл (lifecycle) Service Worker

  1. register
  2. install
  3. activate
  4. fetch

Понимание жизненного цикла (lifecycle) помогает объяснить "почему старая версия еще активна".

Стратегии кэширования

  • Cache-first
  • Network-first
  • Stale-while-revalidate

Одна стратегия на все запросы обычно дает плохой результат.

Offline fallback

  • offline.html вместо "белого экрана"
  • Понятное сообщение для пользователя
  • Контролируемый UX при сетевой ошибке

Где хранить offline-данные

  • localStorage: проще, но ограниченно
  • IndexedDB: гибче и лучше для очередей

Для offline-queue предпочтительнее IndexedDB.

Синхронизация после reconnect

  1. Сохраняем действие offline
  2. Ждем online
  3. Отправляем очередь
  4. Очищаем успешно синхронизированное

Web App Manifest

  • name, short_name
  • start_url, display
  • icons 192 и 512

Manifest — обязательный элемент возможности установки.

Предложение установки (install prompt) и UX

  • Не спамить предложением установки сразу
  • Показывать после понятного действия
  • Объяснять пользу установки

Проверка в DevTools

  • Application -> Service Workers
  • Application -> Manifest
  • Network -> Offline
Проверяем не только online,
но и реальный offline-сценарий.

Lighthouse как sanity-check

  • Быстрая проверка PWA критериев
  • Показывает явные пробелы
  • Не заменяет ручной сценарий
